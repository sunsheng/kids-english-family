# 技术选型调研：发音/释义 API 与开源 K12 词库

> 调研时间：2026-07-01。本报告所有 API 与 GitHub 仓库信息均通过实际搜索、抓取甚至直接发起 HTTP 请求核实（部分接口如有道 `dictvoice`、Free Dictionary API 音频地址已做实测），未凭记忆编造链接。时效性强的信息（价格、免费额度）已标注核实时间，**正式选型前建议对关键接口做一次复测**。

## 第一部分：单词发音 / 释义 API

### 1.1 核心结论

"发音+释义"需求要拆成两类服务：**词典查词类**（释义+音标+例句，最好自带发音）和**发音/TTS类**（单纯把单词读出来）。市面上没有一个免费 API 能同时把两者做到完美——免费的词典 API 普遍存在美式/英式音频不全或无例句的短板；国内 TTS 厂商大多不区分美式/英式；海外厂商（Google）在大陆不可用。

### 1.4 推荐方案

**释义/词典数据：主 + 备双通道**

1. **主力**：有道少儿词典 API（字段最贴合 K12，含年级难度分级）。但**不支持控制台自助开通**，需工作日联系有道商务人工审批报价（电话 010-8255-8901 / 邮件 AIcloud_Business@corp.youdao.com）——**建议尽早启动这一流程**，避免卡项目排期。
2. **备用/交叉验证**：Free Dictionary API（免费无 Key），用于（a）有道审批期间的过渡方案、（b）冷门词兜底、（c）释义准确性交叉校验。缺点是美英双音频不保证齐全，需做空值判断。
3. 预算允许可加：百度智能云"文本翻译-词典版"（唯一同时给出美英双音标+例句+发音的国内平行备选），分散对单一厂商的依赖。

**发音音频：不建议实时调用，建议"预生成 + 自建 CDN 缓存"**

1. 词库中的固定单词（教材词汇、中高考词汇，几千到几万量级）**离线批量预生成** mp3，存入对象存储（阿里云 OSS/腾讯云 COS）+ CDN，前端播放静态资源。来源优先级：有道词典 API 的 `usSpeech`/`ukSpeech` 字段 → 缺口用有道智云 TTS 或 Azure 中国区 TTS 批量补齐。
2. **不建议**把 `dictvoice` 非官方接口作为生产环境唯一依赖，可用于 MVP/demo 阶段，需加缓存+限频。
3. 用户生词本中词典未覆盖的生僻词：走 TTS 实时合成兜底，不实时调用词典 API。
4. Web Speech API 仅作浏览器端最终降级方案（CDN 加载失败时），不作为主力。

## 第二部分：K12 英语词库开源数据源

> **项目当前定位说明**：本产品现阶段为非商用的研究学习项目，因此下文标注"不建议商用"的仓库（`kajweb/dict`、`qwerty-learner`、`lilinji/English`、`maimemo-export`）**均可正常使用**——`lilinji/English` 的 CC BY-NC-SA 许可证本身就是"非商用可用"，其余几个的顾虑也仅在于"商用"层面。这几个来源正好能补上人教版之外的外研版/北师大版教材词表、以及 2019 新课标分单元数据（详见 2.3 节）。**如果未来产品要转向商业化/正式对外发布，需要重新评估这几个来源的版权风险**，届时仍以 `ECDICT` + `DictionaryData` 为可商用底座。

### 2.1 核心发现：法律风险是最大隐患

GitHub 上流传最广的中高考词库（`qwerty-learner`、`kajweb/dict` 系）**数据源头都是爬取自商业背单词 App**，普遍无 License 声明或明确禁止商用，直接用于商业产品有版权风险。真正**许可证清晰、可放心商用**的项目只有两个，建议以它们为基础拼装词库。

### 2.4 缺口的替代建议

> 以下建议适用于未来若产品转向商业化、重新评估版权后仍存在的缺口；在当前非商用研究学习阶段，2.2 节的 `lilinji/English` 与 `maimemo-export` 已可直接覆盖这些缺口，无需走以下流程。

1. **人工整理**（优先做人教版+新课标增补）：购买正版教材，按 Unit 录入 Excel/CSV，用 ECDICT 的释义/音标数据反查节省录入成本，只需人工核对"词属于哪个年级哪个单元"。
2. **购买教辅/题库数据授权**：可接洽新东方、学而思、可可英语、扇贝单词等，洽谈教辅词库/API 授权（未深入核实商务条款，需单独立项调研）。
3. **渐进式覆盖策略（推荐）**：人教版是全国采用最广、且唯一有合规开源分单元数据支撑的版本，**建议 MVP 先只做人教版（小学+初中+高中）+ 中考1600 + 高考3500**，外研版/北师大版留到后续迭代，视用户需求决定自建还是采购。

### 2.5 词库技术方案小结

- 数据底座：`ECDICT`（MIT，中高考标签+释义音标）与 `LinXueyuanStdio/DictionaryData`（Apache-2.0，教材分单元，人教版）按单词文本 JOIN，构建 PostgreSQL 词库表（对应 [`database-schema.md`](./database-schema.md) 中的 `words` / `word_book_entries`）。
- **当前非商用阶段**，补充 `lilinji/English`（外研版/北师大版等18+教材版本）与 `maimemo-export`（2019新课标人教版初中）扩大教材覆盖范围；`kajweb/dict`/`qwerty-learner` 可用于数据结构参考和交叉校验。
- 例句/音频缺口用第一部分的有道词典 API / Free Dictionary API 批量回填。
- **若未来产品转向商业化**，需将词库数据源收敛回 `ECDICT` + `DictionaryData` 这两个许可证清晰的来源，`lilinji`/`maimemo-export`/`kajweb`/`qwerty-learner` 需重新采购授权或替换。
