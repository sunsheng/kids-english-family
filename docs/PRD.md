# 产品需求文档 (PRD)：少儿英语·家庭版 (Web PC)

版本：v1.0
状态：设计中

## 1. 文档概述

- **产品名称**：少儿英语·家庭版（暂定名）
- **产品定位**：一款面向 PC 网页端的家庭英语词汇学习工具，服务于小学至高中的孩子。
- **核心理念**：专注、纯净、科学。无广告、无社交、无游戏、无付费。
- **目标用户**：家长（管理者）+ 小学/初中/高中学生（使用者）。

## 2. 核心功能需求 (MVP)

### 2.1 用户与家庭系统（核心）

| 功能模块 | 功能点        | 详细描述                                                                                                                        |
| -------- | ------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| 账户体系 | 家长注册/登录 | 仅需一个主账号（邮箱/手机号），作为全家的总入口。                                                                               |
| 学员管理 | 学员档案 CRUD | 家长登录后，可新增/编辑/删除学员档案（例如：儿子-小学五年级，女儿-初一），并为每个学员设置发音口音偏好（美式/英式，默认美式）。 |
| 学员切换 | 学习身份切换  | 左侧边栏显示所有学员头像。点击头像，即刻切换为对应孩子的学习上下文。                                                            |
| 数据隔离 | 进度独立      | 每个学员的词库进度、生词本、学习天数完全独立，互不干扰。                                                                        |

### 2.2 词库与计划

| 功能模块 | 功能点        | 详细描述                                                   |
| -------- | ------------- | ---------------------------------------------------------- |
| 词库列表 | 教材同步      | 覆盖小学、初中、高中主流教材（人教版、外研版等）单元词汇。 |
| 词库列表 | 考纲词汇      | 小学核心词、中考 1600 词、高考 3500 词。                   |
| 学习计划 | 选书学习      | 选择一本词书 -> 设置每日新词量（10/20/30） -> 开启计划。   |
| 学习状态 | 进行中/未开始 | 首页显示当前正在学习的词书及进度。                         |

### 2.3 学习流程（PC 优化）

| 功能模块 | 功能点      | 详细描述                                                                                                     |
| -------- | ----------- | ------------------------------------------------------------------------------------------------------------ |
| 单词认知 | 翻转卡片    | 正面显示单词，空格键翻转显示释义、音标、例句（全程键盘操作，无需鼠标）。                                     |
| 发音     | 自动发音    | 卡片出现时自动播放发音，无需点击；按 R 键可重新播放。美式/英式口音在学员设置中统一切换，不在卡片上重复选择。 |
| 交互决策 | 认识/不认识 | 方向键 ←（不认识，加入生词本）/ →（认识，跳过）。                                                            |
| 拼写测试 | 三轮巩固    | 详见下方 2.3.1，独立入口/模式，不与认知卡片合并。                                                            |

#### 2.3.1 拼写测试：三轮巩固设计

拼写测试的目标不是"测一次对错"，而是通过"听 3 遍、写 3 遍、难度递增"强化记忆，具体规则：

1. **第 1 轮（少量挖空 + 提示）**：挖空单词中一小段连续字母（长度通常 1~~3 个字母，首字母固定保留作为锚点，如 apple 显示 `a__` + `ple` 中挖空 "le"）。下方展示打乱顺序的字母提示块（正确项 + 2~~3 个形近干扰项，如挖空 "le" 时提示块包含 "le"、"ll"、"el"）。提示块**仅供视觉参考，不可点击**，作答仍需通过键盘逐格输入。
2. **第 2 轮（换位置挖空，无提示）**：挖空长度与第 1 轮接近，但换成不同位置的连续字母，不再显示提示块，靠记忆默写。
3. **第 3 轮（全部挖空，无提示）**：整词挖空，完全默写。

其他规则：

- 每轮开始时自动播放一次发音（对应"听 3 遍写 3 遍"）。
- 答错：标红 + 清空当前轮的挖空格，原地立即重试，直到答对才能进入下一轮或下一个词（拼写错误绝不跳过）。
- 三轮全部通过，这个词才算完成，"今日进度 X/20" 才 +1；学习中额外展示"第 n/3 轮"的小进度提示。
- 详细的挖空算法、干扰项生成规则、键盘字母格交互规范，见 [`spelling-drill-design.md`](./spelling-drill-design.md)。

### 2.4 复习与统计

| 功能模块 | 功能点   | 详细描述                                           |
| -------- | -------- | -------------------------------------------------- |
| 智能复习 | 遗忘曲线 | 根据艾宾浩斯算法，自动生成每日复习任务（旧词）。   |
| 生词本   | 集中攻克 | 单独页面展示所有标记过的生词，支持单独听写或浏览。 |
| 数据统计 | 学习日历 | 展示连续打卡天数（热力图）。                       |
| 数据统计 | 成就展示 | 显示累计学习词汇量、今日已学数量。                 |

## 3. 页面结构设计 (Page Structure)

采用左右布局，左侧导航常驻，右侧内容区切换。

```
+-------------------------------------------------------+
|  Header (顶部栏)                                       |
|  当前学员：小明 [切换] | 今日进度：20/20 | 退出          |
+------------------------+------------------------------+
|                        |                              |
|  Sidebar (左侧导航栏)   |  Main Content (右侧主内容区)   |
|  ---------------------  |  ----------------------------|
|  [家庭学员列表]          |                              |
|  > 小明 (小学)          |  [页面A: 仪表盘 Dashboard]     |
|  > 小红 (初中)          |     - 欢迎语 + 今日任务卡片     |
|                        |     - 连续打卡日历             |
|  [选词库]               |     - 快速开始按钮             |
|  [开始学习]             |                              |
|  [复习中心]             |  [页面B: 学习室 Learning]      |
|  [生词本]               |     - 进度条                  |
|  [统计]                |     - 巨大单词卡片 (Word Card) |
|                        |     - 发音按钮                 |
|                        |     - 认识/不认识/拼写按钮      |
|                        |                              |
+------------------------+------------------------------+
```

### 页面详细说明

1. **仪表盘 (Dashboard)**：登录后的首页。核心看板，无多余信息。
2. **选词库 (Library)**：网格化展示词书封面，点击后弹出"设置每日词数"的模态框。
3. **学习室 (Learning Room)**：核心交互页。极简设计，全屏只有一个单词卡片和操作区。
4. **复习中心 (Review)**：列表展示待复习单词，操作逻辑同学习室。
5. **生词本 (Vocabulary)**：表格形式展示历史生词，支持搜索和删除。

## 4. UI/UX 设计规范 (Style Guide)

### 4.1 色彩规范 (Colors)

- 设计原则：护眼、柔和、低饱和度。
- 背景色：`#F5F7FA`（极浅灰蓝，保护视力）
- 主色调 (Primary)：`#5D9CEC`（柔和钴蓝，用于主要按钮、选中状态）
- 辅助色 (Success)：`#A0D468`（青柠绿，用于"认识"、"已完成"）
- 警示色 (Danger)：`#ED5565`（珊瑚红，用于"不认识"、"错误"）
- 文字色：
  - 主文字：`#333333`（深灰）
  - 次文字：`#999999`（浅灰）

### 4.2 字体规范 (Typography)

- 字体家族：`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, "PingFang SC", "Microsoft YaHei"`
- 字号层级：
  - 单词 (H1)：56px - 72px（超大字号，核心视觉焦点）
  - 释义 (Body)：24px
  - 例句 (Caption)：18px，颜色浅灰
  - 按钮文本：20px，加粗

### 4.3 组件风格 (Components)

- **卡片 (Card)**：圆角 12px，白色背景，极淡阴影 `0 2px 10px rgba(0,0,0,0.05)`。
- **按钮 (Button)**：
  - 高度：56px
  - 圆角：8px
  - 状态：Hover 时颜色加深，Active 时轻微缩放 (scale 0.98)。
- **间距**：大面积留白，不要让内容挤在一起。

### 4.4 交互设计 (Interaction)

- **适用范围**：以下"全键盘操作"规则针对学习室、复习中心、拼写测试等沉浸式学习页面；仪表盘、选词库、生词本等管理类页面仍是常规鼠标+键盘操作，不做额外的全局键盘导航。
- 鼠标：所有可点击元素必须有 hover 变色效果。
- 键盘（学习页面内，全程无需鼠标）：
  - Space（空格）：翻转卡片 / 下一个。
  - Enter：提交拼写 / 确认，**拼写测试中仅答案正确才会跳转到下一轮/下一个词**。
  - Arrow Left/Right：切换"认识/不认识"（仅认知卡片模式生效）。
  - **R：重新播放发音**（认知卡片、拼写测试均可用）。
- 发音：卡片出现、拼写测试每轮开始时自动播放，无需点击；美式/英式口音为学员级设置项（默认美式），不在卡片上逐次切换。
- 动画：拒绝花哨。仅保留 0.2s 的淡入淡出 (fade-in/out) 或位移 (slide)。

## 5. 非功能性需求

- **兼容性**：Chrome 80+, Edge 80+, Firefox 最新版。
- **响应式**：仅针对 PC 端（宽度 > 1024px），无需适配移动端。
- **数据安全**：家长账号拥有最高权限，可管理所有学员数据。

## 6. 技术选型（已确认）

- **前端**：Next.js + TypeScript
- **后端**：Node.js（NestJS/Express）
- **数据库**：PostgreSQL

## 7. 开发计划 (Roadmap)

### Phase 1: 基础设施与静态页面（Week 1）

1. 搭建前端框架（Next.js）。
2. 实现左侧导航栏和顶部栏布局。
3. 实现"仪表盘"页面（静态数据）。
4. 实现"选词库"页面（写死几本教材数据）。
5. 重做 `word-card-mockup.html`，落地新的键盘全流程交互（自动发音、认知卡片方向键、拼写测试三轮布局），作为 Phase 2 开发的静态参照。

### Phase 2: 核心逻辑与数据（Week 2）

1. 接入后端 API（Node.js）。
2. 实现用户登录和学员增删改查，含学员发音口音偏好设置（`students.preferred_accent`）。
3. 实现单词卡片展示和自动发音播放（调用免费词典 API，按学员口音偏好选择美式/英式音频），支持 R 键重新播放。
4. 实现学习进度存储（记录学会了哪些词）。

### Phase 3: 复习算法与闭环（Week 3）

1. 编写艾宾浩斯复习算法（简化版 SM-2，核心难点，但逻辑固定）。
2. 实现"复习中心"和"生词本"。
3. 实现拼写测试三轮巩固功能：挖空算法、干扰项生成、键盘字母格交互，并按"整词三轮是否零错误"判定 SM-2 答对/答错（详见 [`spelling-drill-design.md`](./spelling-drill-design.md)）。
4. 完善数据统计（日历、词汇量）。

### Phase 4: 测试与部署（Week 4）

1. 录入/校验小学到高中的词库数据（CSV 导入）：以 `ECDICT`（中高考词表）+ `DictionaryData`（人教版分单元）为底座，当前非商用研究学习阶段可补充使用 `kajweb/dict`、`qwerty-learner`、`lilinji/English`、`maimemo-export` 覆盖外研版/北师大版和新课标缺口（详见 [`dictionary-and-wordbank-research.md`](./dictionary-and-wordbank-research.md)）。
2. 内部测试（找孩子试用，看操作顺不顺手，重点验证全键盘操作流程是否顺畅）。
3. 部署上线（Vercel/Netlify 前端 + 云服务器后端）。
