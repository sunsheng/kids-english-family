# 拼写测试：三轮巩固设计

对应 [`PRD.md`](./PRD.md) §2.3.1。目标：不是"测一次对错"，而是通过"听 3 遍、写 3 遍、难度递增"强化记忆。拼写测试必须放在独立“测试”页面/模式中，不与“开始学习”的认知卡片合并。全程键盘操作，不依赖鼠标。

## 1. 三轮流程总览

| 轮次    | 挖空范围                                    | 提示                                        | 目的                         |
| ------- | ------------------------------------------- | ------------------------------------------- | ---------------------------- |
| 第 1 轮 | 一小段连续字母（长度 1~3），首字母固定保留  | 有（正确项 + 干扰项，打乱顺序，仅视觉参考） | 借助提示入门，建立"形状记忆" |
| 第 2 轮 | 另一段不同位置的连续字母，长度与第 1 轮接近 | 无                                          | 检验脱离提示后是否记住       |
| 第 3 轮 | 整词全部挖空                                | 无                                          | 纯默写，检验完全记忆         |

以 **apple**（5 字母）为例：

- 第 1 轮：显示 `a p p _ _`，挖空 "le"，下方提示块（打乱）：`le`、`ll`、`el`。
- 第 2 轮：显示 `a _ _ l e`，挖空 "pp"（换了位置），无提示。
- 第 3 轮：显示 `_ _ _ _ _`，整词默写，无提示。

每轮开始时自动播放一次发音（对应"听 3 遍写 3 遍"）；播放使用学员设置里的口音偏好（`students.preferred_accent`），按 **Arrow Up** 键可重新播放（测试页面不用字母 R，避免和挖空作答里的字母 r 冲突）。

## 2. 挖空长度与位置算法

**挖空长度 k**（第 1、2 轮通用）：

```
k = clamp(round(word.length / 3), 1, 3)
```

即短词挖 1 个字母，中等长度词挖 2 个，长词最多挖 3 个，避免长词第 1 轮就挖太多打击信心。

**位置选择**：

- 候选起始位置范围：`[1, word.length - k]`（下标从 0 开始，index 0 即首字母永远保留、不参与挖空）。
- 第 1 轮：从候选位置中随机选一个，记为 `chunk1`。
- 第 2 轮：从候选位置中排除 `chunk1` 后随机选一个，记为 `chunk2`（要求与 `chunk1` 不完全重合）。
- **边界情况**：若单词很短，候选位置只有 1 个（无法选出与第 1 轮不同的位置），则第 2 轮改为：先尝试把 `k` 减 1 后重新找候选位置；仍找不到不同位置时，第 2 轮直接复用第 1 轮的挖空位置（词太短、无法进一步区分，属于可接受的降级）。
- 第 3 轮：`chunk3` = 整个单词（`[0, word.length - 1]`）。

## 3. 提示块（干扰项）生成规则 —— 仅第 1 轮

给定第 1 轮正确挖空片段 `correct`（长度 k），生成 2~3 个同长度的干扰项，规则：

1. **相邻字母互换**（k ≥ 2 时可用）：交换 `correct` 内相邻两个字母，如 "le" → "el"。
2. **形近字母替换**：从下表选一对常见易混字母，替换 `correct` 中的一个字母，生成同长度的新字符串：
   - 元音混淆：a/e、e/i、i/y、o/u
   - 辅音混淆：c/k、s/z、f/v、m/n、b/d
   - 单双写混淆：将 `correct` 中某个字母替换为其双写或去重（如 "ll" → "l" 需配合长度调整，仅在 k ≥ 2 且结果仍保持长度 k 时采用，否则跳过）
3. 生成后**去重**（不能与 `correct` 相同，干扰项之间也不能重复）；若某种规则生成失败（如 k=1 时无法做相邻互换），跳过该规则，用形近字母替换规则补足数量。
4. **兜底规则**：若正确片段是重复字母（如 "pp"）或包含不在形近替换表里的字母（如 "x"），规则 1、2 可能一个干扰项都生成不出来。此时用随机替换兜底——随机选一个位置，换成不同的随机字母，重复直到凑够数量。保证提示块任何情况下都不会只剩正确答案一个选项。
5. 最终提示块 = `[correct, distractor1, distractor2, (可选distractor3)]`，**整体打乱顺序**后展示。
6. 提示块仅做视觉参考展示（不可点击、不参与 Tab 焦点），作答只能通过键盘在挖空格中逐格输入。

## 4. UI / 键盘交互规范

- “开始学习”页面只做认知学习：看单词、听发音、翻卡、方向键判断认识/不认识，不出现字母格输入。
- “测试”页面才渲染以下字母格和键盘输入流程；复习中心进入到期词测试时也复用测试页面的交互。

- 单词以"字母格"逐格渲染：已知字母显示为静态字符（浅底深字，不可编辑）；挖空字母显示为空白输入格（默认边框浅灰 `#E4E9F0`，聚焦时边框变主色 `#5D9CEC`）。
- 轮次开始时，焦点自动落在第一个挖空格。
- 输入一个字母后自动跳到下一个挖空格（跳过已知字母格）；Backspace 清空当前格并把焦点移回上一个挖空格。
- 挖空格全部填满后，按 **Enter** 提交校验（不需要额外按钮）。
- 校验结果：
  - **全部正确**：挖空格边框/底色短暂变为成功色 `#A0D468`（0.2s 淡入淡出），随后自动进入下一轮（或下一个词），并自动播放下一轮/下一个词的发音。
  - **存在错误**：出错的格子边框/底色短暂变为警示色 `#ED5565`，随后**清空本轮所有挖空格**（不清空已知字母），焦点重新回到第一个挖空格，原地立即重试，不跳转、不计入"下一个"。
- 全程不出现需要点击的按钮或提示块，符合"学习时不用鼠标"的要求。

## 5. 与复习算法（SRS）的联动

判定规则见 [`database-schema.md`](./database-schema.md) §4：一个词的"答对/答错"以**三轮是否全程零错误**为准，而不是按每次按键提交单独判定——只要任意一轮出现过错误重试，这个词本次就按"答错"计入 SM-2（`ease_factor` 下降、进入生词本）；三轮全部一次通过才算"答对"。`times_correct`/`times_wrong` 仍按实际提交次数原样累加，仅用于展示统计，不影响 SRS 分支选择。

## 6. 待实现时确认的细节

- 提示块的干扰项生成目前是规则算法（非 AI 生成），如果实际效果不理想（比如干扰项过于离谱或过于像正确答案），后续可以调整形近字母表或加权随机。
- 第 2 轮"位置不同"目前只保证起始下标不同，不保证与第 1 轮完全不重叠；如果测试后发现体验上需要"完全不重叠"，可以在候选位置筛选时加一条排除重叠区间的规则。
